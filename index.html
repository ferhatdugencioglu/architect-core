ARCHITECT CORE — User Guide
📌 What This Tool Is NOT
Architect Core is not a ticket management system.
It is not designed for daily open/close ticket workflows.
Instead, it is used for: archiving major issues, root cause analysis, and preserving solutions.
📚 CASE LIBRARY
A structured archive for major problems and their solutions.

Problem: What happened? User complaint or system error.
Environment: Server, OS, version, site where the issue occurred.
Error / Log: System error message or log output.
Root Cause: The actual underlying cause.
Solution: How was it resolved?
Commands: Key commands used.
Workaround: Temporary fix or reference file path.
Notes: Expert notes, references, contact info.
📋 WORK LOG / CASE HISTORY
A chronological work log inside each case.

Entry types:
📝 Note — General note
🔌 Engineer Connected — Engineer access log
📨 Vendor Support Reply — Vendor/TAC response
🔧 Action Taken — Action performed
🔍 Finding — Discovery or observation
🩹 Temporary Fix — Workaround applied
✅ Permanent Fix — Root fix applied
📊 Monitoring Update — Observation after fix
🏁 Resolved — Issue confirmed resolved

To add an entry: Select type → Enter title + description → + ADD → UPDATE CASE to save.
📝 TECH NOTES
Quick storage for commands, scripts, and technical notes.
Organized by categories.
Supports file upload (TXT, PS1, PDF).
Global search scans all notes.
🗑 RECYCLE BIN
All deleted items go to the Recycle Bin.
RESTORE returns an item to its original location.
RESTORE TO moves it to a different category.
DELETE ALL permanently clears the bin.
